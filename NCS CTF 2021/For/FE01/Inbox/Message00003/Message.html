We are enhancing Core eDiscovery, Advanced eDiscovery legal holds, Information Governance and Records Management retention policies by:

  • Improving scalability and performance to reduce the number of hold distribution errors and to support growing number of tenants onboarding to service. 
  • Locking a pending hold or pending policy from updates (e.g. cancelling, deleting, editing), resulting in more reliable and predictable service.

Key points

  • Timing: March 29, 2021
  • Roll-out: tenant level
  • Action: review and assess

How this will affect your organization:

Following this roll out, organizations should start seeing less holds errors and retention policy (distribution) errors.

What you need to do to prepare:

For most organizations there is no action required. If you're organization is not using Core eDiscovery, Advanced eDiscovery legal holds, Information Governance and Records Management retention policies, you will not be affected.

There are rare instances where the system cannot automatically fix an error related to active users that were placed on hold, then turned inactive. In this case, you may run into this new error "We can't identify this user with the existing identifiers. The link between this user and the policy is broken."

  • To fix this issue, please review how to Troubleshooting common eDiscovery issues.
